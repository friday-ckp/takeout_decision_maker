# 头脑风暴记录 - 外卖点餐决策器

**日期：** 2026-03-26
**技法：** What If Scenarios + Reversal Inversion
**平台定位：** Web 端（本地部署）

---

## 核心问题

**痛点：** 每天中午点外卖时选择困难，在平台上刷太久，浪费时间，想快速做出决定。

**设计原则：** 帮用户快速结束决策，而不是延长使用时长。

---

## 创意清单

### 决策交互

| 功能 | 描述 |
|------|------|
| 🎡 **转盘决策** | 一键旋转，停哪吃哪，极简决策入口 |
| 🔄 **转前询问** | 每次转动前询问"要加新选项吗？"，否则按历史自动填充转盘 |
| 🔢 **限制重转** | 限制每日重转次数，逼用户接受结果，避免无限纠结 |
| 🙈 **盲选模式** | 不显示结果直接确认，作为可选模式 |
| 💣 **扫雷玩法** | 盲点格子，第一个"炸"到的就是今天吃的 |

### 个性化与记忆

| 功能 | 描述 |
|------|------|
| 😊 **心情选择器** | 右上角心情 icon，每天点一次表情，影响推荐结果 |
| 🍜 **口味偏好** | 今天想吃咸/甜、重口/清淡，比心情更精准的过滤条件 |
| 📅 **历史排除** | 自动记录最近吃过的，推荐时自动降低权重或排除 |
| 📊 **饮食规律洞察** | 记录点餐时间、口味偏好，长期后能识别规律（如"每周三喜欢川菜"） |

### 餐厅管理

| 功能 | 描述 |
|------|------|
| ➕ **添加餐厅** | 手动添加常点餐厅到候选池 |
| ⭐ **收藏** | 标记喜欢的餐厅 |
| 🚫 **踩雷拉黑** | 吐槽入口 → 直接永久拉黑该店，不再出现 |
| 🔍 **智能过滤** | 最多显示 5 个候选，不是全量展示 |

### 多人协作

| 功能 | 描述 |
|------|------|
| 🔗 **分享链接** | 发起人创建"决策房间"，生成唯一链接分享给朋友 |
| 👥 **实时多人决策** | 所有人打开链接后各自参与，实时汇总揭晓结果 |
| 🔌 **技术方案** | WebSocket 实现实时同步，房间 ID 通过链接参数传递 |

### 体验设计原则

| 原则 | 说明 |
|------|------|
| ⚡ **打开即决策** | 首屏直接显示决策界面，零干扰，零跳转 |
| 🚫 **零广告** | 没有推荐流、Banner、活动弹窗 |
| 🙅 **隐藏菜单** | 决策阶段只看店名/品类，不看高清菜单图（防止越看越纠结） |
| 📱 **单一核心动作** | 整个产品围绕"做决定"这一个动作设计，其他功能收起 |

---

## 核心价值主张

> **这个 Web 端不是外卖平台，是一个帮你在 30 秒内结束"吃什么"这个问题的决策工具。**
