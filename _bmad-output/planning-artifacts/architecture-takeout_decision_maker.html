03 · 多人决策 WebSocket 会话流
sequenceDiagram participant H as 🧑 发起人 (Host) participant S as ⚙️ Server participant DB as 🗄️ MySQL participant G as 👥 受邀者 (Guests) H->>S: POST /sessions (创建会话, 选择模式) S->>DB: INSERT decision_sessions\n+ candidate_snapshot(快照) S-->>H: { share_token, lobby_url } H->>S: WS connect /session/:token S-->>H: event: session_state(waiting, participants:[host]) G->>S: GET /session/:token (输入昵称) S->>DB: INSERT users(临时) + session_participants S-->>G: WS connect 成功 S-->>H: event: participant_joined { name } S-->>G: event: session_state(waiting, participants:[...]) H->>S: WS event: start_deciding S->>DB: UPDATE decision_sessions SET status=deciding S-->>H: event: deciding_started S-->>G: event: deciding_started Note over H,G: 扫雷模式 — 先到先得 G->>S: WS event: cell_clicked { cell_index } S->>S: 检查是否第一个有效点击\n(原子操作，忽略后续点击) S->>DB: UPDATE decision_sessions\nSET result_restaurant_id S-->>H: event: result_revealed { restaurant } S-->>G: event: result_revealed { restaurant } Note over H,G: 发起人确认结果 H->>S: WS event: confirm_result S->>DB: INSERT decision_history\nUPDATE status=done S-->>H: event: session_done S-->>G: event: session_done (跳转结束页)
